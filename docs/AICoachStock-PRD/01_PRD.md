> 생성일: 2026-05-13
> 생성 도구: Show Me The PRD
> 대상 기기: Galaxy Z Fold 7 (Android 15, 12GB RAM, Snapdragon 8 Elite for Galaxy)

## 1. 제품 개요

### 한 줄 요약
나만의 매매 원칙을 기억하고, 매매일지를 복기시켜주며, 진입·청산 규율을 코칭하는 **온디바이스 AI 주식 트레이닝 코치 앱**.

### 해결하는 문제
- 매매 후 감정적으로 휘둘려서 같은 실수를 반복하는데 **객관적으로 짚어줄 사람이 없다**
- 진입 시점에 본인이 정한 원칙을 **잊거나 무시**하고 사후에 후회한다
- 손절·익절 라인을 정해놔도 **실시간 감시·환기**가 없으면 무너진다
- 종목·재무 정보를 조사할 때 **수동 검색·정리가 너무 번거롭다**
- 외부 GPT 서비스는 **개인 매매 데이터·계좌 정보를 클라우드에 올리기 부담**스럽다

### 핵심 가치
1. **온디바이스 AI (Gemma 4 E4B)** — 매매 데이터, 일지, 원칙이 폰 밖으로 나가지 않음
2. **개인 맞춤 코치** — 일반 챗봇이 아니라 *내 원칙·내 매매기록*을 컨텍스트로 받는 코치
3. **시세 연동 코치** — 한투 OpenAPI로 실시간 시세 받아 손절·익절 라인 자동 감시
4. **한국·미국 시장 동시 지원** — 한투 국내 + 한투 해외주식 API로 한 앱에서 관리

### 차별점 (vs. 기존 대안)
| 대안 | 한계 | 본 앱의 차별 |
|------|------|------------|
| ChatGPT/Claude | 클라우드 전송, 매매기록 컨텍스트 부재 | 온디바이스, 내 일지·원칙을 RAG로 주입 |
| 증권사 앱 (한투 MTS 등) | 매매 실행 도구일 뿐, 코칭 X | 코치 역할 특화 |
| 매매일지 앱 (Stock Journal 등) | 단순 기록·통계, AI 분석 부재 | LLM 기반 패턴·감정 분석 |
| 자동매매 봇 | 시그널 생성형, 규율 학습 안 됨 | 인간 의사결정 보조에 집중 |

## 2. 사용자

### 주요 사용자
- **누구**: 본인 (1인, 개인 사용. 추후 가족·지인 확장 가능성 열어둠)
- **상황**:
  - 출퇴근 지하철에서 관심종목 시세 확인하며 진입 검토
  - 점심·저녁 짬에 오전·당일 매매 복기
  - 주말에 한 주 매매일지 전체 리뷰
  - 미장 개장 시간(밤 11시~)에 미국 종목 진입 결정
- **목표**:
  - 같은 실수를 반복하지 않기
  - 감정 매매 줄이기
  - 손절·익절 규율 지키기
  - 종목 리서치 시간 줄이기

### 사용자 시나리오

**시나리오 1 — 진입 전 체크리스트**
1. 관심종목 리스트에서 *삼성전자* 클릭
2. "지금 진입 고려 중" 버튼 → 진입 체크리스트 표시
3. 미리 등록해둔 원칙(시총·PER·차트 위치·뉴스 확인 등) 질문에 답변
4. Gemma 4 E4B가 답변과 시세를 보고 GO / HOLD / STOP 의견 + 근거 제시
5. 결정을 매매기록에 자동 기록 (실제 주문은 별도 증권사 앱에서)

**시나리오 2 — 매매 후 복기**
1. 매매 후 매매기록 입력 (종목·가격·수량·매매 이유·감정 태그)
2. "AI 복기 요청" 클릭
3. Gemma 4 E4B가 *내 매매 원칙*과 대조해서 위반 여부, 감정 매매 가능성, 교훈 도출
4. 내 메모와 함께 일지로 저장 → 주말 리뷰 시 패턴 확인

**시나리오 3 — 손절 알림**
1. 매수 시 손절가·익절가 설정 → 해당 종목을 WebSocket 구독 대상에 추가
2. 백그라운드 Foreground Service가 한투 WebSocket 연결 유지 (장 시간대)
3. 체결 tick이 손절가 도달 시 즉시(<1초) 알림 + AI 메시지: "원칙대로 손절 검토. 감정 매매 주의."
4. 사용자 확인 후 별도 증권사 앱에서 청산
5. 비장시간엔 WebSocket 종료, REST로 종가 1회 표시

**시나리오 4 — 종목 리서치 Q&A**
1. *엔비디아* 검색 → 기본 종목 정보(시총·섹터·52주 고저)
2. "이 종목 어떻게 생각해?" 자유 채팅
3. Gemma 4 E4B가 *내가 입력한 재무 데이터·뉴스 요약*과 *내 투자 원칙*을 결합해 답변
4. (주의: 모델 학습 컷오프 한계로 최신 실적은 수동 입력 필요)

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 매매 원칙 CRUD | 진입/청산/자금관리/심리 카테고리별 규칙 관리 | P1 (MVP) | 간단 |
| 매매기록 입력 | 종목·가격·수량·이유·감정 태그 수동 입력 | P1 (MVP) | 간단 |
| 모델 자동 다운로드 | 첫 실행 시 Gemma 4 E4B `.task`(~3GB) 다운로드·검증·캐시 | P1 (MVP) | 보통 |
| AI 일지 복기 | Gemma 4 E4B로 매매를 원칙과 대조해 분석 | P1 (MVP) | 복잡 |
| 코치 채팅 | 자유 형식 대화. 세션·메시지 영속화 | P1 (MVP) | 보통 |
| 진입 체크리스트 | 원칙 기반 질문 응답 후 AI 판정 | P1 (MVP) | 보통 |
| 종목 리서치 Q&A | 종목 메타·수동 재무 입력 + AI 질의응답 | P1 (MVP) | 보통 |
| 한투 OpenAPI 시세 (WebSocket) | 국내+해외 실시간 체결 시세 스트림. 장 시간대 상시 연결, 비장시간 REST 종가 폴백 | P1 (MVP) | 복잡 |
| 손절/익절 알림 | Foreground Service로 WebSocket 유지 + 틱 단위 즉시 감지 + 알림 | P1 (MVP) | 복잡 |
| 수익 대시보드 | 일·주·월 수익률, 승률, 평균 보유 기간 | P2 | 보통 |
| 감정 태그 트렌드 | 감정 태그별 손익 상관관계 시각화 | P2 | 보통 |
| 원칙 위반 패턴 리포트 | 가장 자주 위반한 원칙 자동 추출 | P2 | 보통 |
| 음성 입력 | 매매 이유·메모 음성 입력 (Android STT) | P3 | 보통 |
| 원칙 자동 개선 제안 | AI가 위반 패턴 보고 원칙 수정안 제안 | P3 | 복잡 |
| 키움 OpenAPI 이중화 | 한투 장애 시 폴백 | P3 | 복잡 |
| 클라우드 백업 | 사용자 선택형, E2EE | P3 | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름

```
[최초 진입]
  -> 한투 API키 등록 (1회)
  -> 매매 원칙 초기 입력 (5~10개)
  -> 메인 대시보드

[일상 사용]
  -> 메인 -> 관심종목/검색 -> 종목 상세
  -> [경로A: 진입 검토] 진입 체크리스트 -> AI 판정 -> 매매기록 저장
  -> [경로B: 복기]       매매기록 입력 -> AI 복기 -> 일지 저장
  -> [경로C: 채팅]       코치 채팅 -> 컨텍스트 자유 질의
  -> [경로D: 감시]       (백그라운드) 손절·익절 라인 도달 -> 푸시 알림
```

### 상세 흐름

1. **온보딩**
   - 면책 동의(투자 책임은 사용자 본인)
   - **모델 다운로드 단계**: 앱 내부 저장소에 `gemma-4-e4b-it.task` 존재 여부 확인 → 없으면 Wi-Fi 권장 안내 후 다운로드 (진행률·일시정지·재개 지원, SHA-256 무결성 검증)
   - 한투 API키 등록
   - 매매 원칙 가이드(예시 템플릿 제공)
   - 모델 로드 테스트 추론 (Hello prompt)

2. **메인 대시보드**
   - 오늘 매매 요약, 관심종목 시세, 활성 알림(손절·익절 라인 임박), 최근 복기

3. **종목 상세**
   - 시세·차트(간단) + 진입 체크리스트 진입 + 코치 채팅 진입 + 매매기록 진입

4. **AI 작업 흐름** (공통)
   - 사용자 입력 → 컨텍스트 수집(원칙·관련 매매기록·시세) → Gemma 4 E4B 추론 → 결과 표시 + DB 저장

## 5. 성공 기준

- [ ] **사용 빈도**: 평일 기준 주 5회 이상 앱 열기 (8주 이상 유지)
- [ ] **복기 커버리지**: 실제 체결 매매 중 80% 이상에 AI 복기 기록 존재
- [ ] **원칙 적용도**: 진입 체크리스트 미사용 매수 비율이 1개월 후 20% 이하
- [ ] **알림 효과**: 손절 알림 발생 후 5분 이내 사용자 반응(확인·청산·연기) 90%
- [ ] **응답 속도**: AI 복기 1건 생성 평균 응답 시간 ≤ 15초 (Z Fold 7 기준)
- [ ] **시세 반응성**: 손절가 체결 발생 → 알림 표시까지 평균 ≤ 2초 (WebSocket 기준)
- [ ] **데이터 안전**: 폰 분실/초기화 외 시나리오에서 매매기록 손실 0건
- [ ] **개인적 KPI**: 사용 3개월 후 본인 평가 "감정 매매 줄었다" 자기 평점 ≥ 7/10

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- **자동 주문 실행** — 이유: 자동매매 사고 리스크, 인증 복잡도, 규율 학습 목적과 상충
- **실시간 차트 (분봉/체결강도/호가창)** — 이유: 본 앱은 코치 역할. 차트는 증권사 MTS에서 보기
- **백테스트 엔진** — 이유: 과거 데이터 확보·성능 부담. P3 이후 검토
- **다른 사용자와의 소셜 기능 (포트폴리오 공유 등)** — 이유: 개인 코치 컨셉 훼손, 운영 부담
- **종목 시그널/추천 푸시** — 이유: LLM은 신호 생성에 부적합, 책임 리스크
- **iOS 버전** — 이유: 본 사용자는 Android 단일 기기. KMP 전환은 P3 이후
- **소셜 로그인·계정 시스템** — 이유: 개인 앱, 단일 기기 사용
- **암호화폐·선물·옵션** — 이유: 현물 주식에 집중
- **유료 결제·구독** — 이유: 개인 사용 앱

## 7. [NEEDS CLARIFICATION]

> 아직 결정되지 않은 사항. 개발 전에 정해야 합니다.

- [ ] **한투 OpenAPI 인증 방식** — REST(Access Token) vs. 오픈API 신청 방식. 둘 다 가능하지만 모바일에서는 REST + Token Refresh가 일반적
- [ ] **WebSocket 구독 한도 (41종목)** — 관심종목 + 활성 PriceAlert 종목 합쳐 초과 시 우선순위 정책 (활성 알림 우선, 그다음 관심종목 최근순)
- [ ] **국내/해외 동시 연결** — 한투 WebSocket이 1 연결에 국내+해외 TR 혼합 구독 허용하는지, 별도 연결 필요한지 확인
- [ ] **미장 시세 무료 범위** — 한투 해외주식 API의 실시간 vs. 지연(15분) 차이 검증 필요
- [ ] **모델 다운로드 호스트 URL** — Hugging Face(`google/gemma-4-e4b-it-litert`) vs. Kaggle Models 중 어디서 받을지, 라이선스 동의 토큰 필요 여부
- [ ] **다운로드 실패 폴백** — 네트워크·디스크 부족 시 처리. Gemma 3n E4B(~2GB) 폴백 자동 제안 여부
- [ ] **MediaPipe LLM Inference API 최신 버전 호환** — Gemma 4 E4B 지원 시점 확인
- [ ] **컨텍스트 길이 전략** — E4B 컨텍스트 윈도우 안에서 어떤 매매기록을 RAG로 주입할지 (최근 N건 vs. 유사 종목 N건)
- [ ] **감정 태그 분류 체계** — 자유 입력 vs. 정해진 enum(공포/탐욕/FOMO/확신/혼란 등)
- [ ] **알림 정책** — 손절가 도달 외에 추가 알림(원칙 위반 의심 매매 직후 등) 정의 필요
